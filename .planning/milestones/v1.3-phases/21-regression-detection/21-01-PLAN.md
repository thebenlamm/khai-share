  <name>Task 1: Create RegressionDetector module</name>
  <files>src/agent/regressionDetector.js</files>
  <behavior>
    - detectRegressions(baseline, currentPages) returns { hasRegressions, summary, regressions }
    - regressions is an array of { type, url, before, after, threshold? }
    - type values: 'title_changed', 'page_missing', 'page_new', 'status_changed', 'timing_regression'
    - title_changed: before = baseline title, after = current title (only when BOTH are non-null and differ)
    - page_missing: url present in baseline.snapshot.pages but absent in currentPages
    - page_new: url present in currentPages but absent in baseline.snapshot.pages
    - status_changed: status code differs between baseline and current for same URL
    - timing_regression: current loadTime > baseline.thresholds.pageLoadTime
    - summary includes counts: { titleChanges, missingPages, newPages, statusChanges, timingRegressions, total }
    - hasRegressions is true when total > 0
    - URL matching is exact string equality (no normalization — consistent with how crawler records URLs)
    - If baseline or currentPages is null/empty, return { hasRegressions: false, summary: { total: 0, ... }, regressions: [] }
    - Function is pure — no I/O, no side effects, no requires beyond what it needs
  </behavior>
  <action>
    Create src/agent/regressionDetector.js with a single exported function detectRegressions(baseline, currentPages).

    Build a URL-keyed Map from baseline.snapshot.pages for O(1) lookup. Build a separate Set of baseline URLs and current URLs for missing/new page detection.

    Detection order:
    1. For each baseline page: check if URL appears in currentPages (missing detection)
    2. For each current page: check if URL appears in baseline pages (new page detection)
    3. For shared URLs: compare title, status, and loadTime

    Title change rule: only flag if BOTH baseline title AND current title are non-null strings that differ. If either is null, skip — null means title was not captured.

    Timing rule: use baseline.thresholds.pageLoadTime as the ceiling. Flag if currentPage.loadTime > threshold. Do NOT compare current vs baseline loadTime directly — threshold is the contract, not the snapshot value.

    Return shape:
    ```js
    {
      hasRegressions: boolean,
      summary: {
        titleChanges: number,
        missingPages: number,
        newPages: number,
        statusChanges: number,
        timingRegressions: number,
        total: number
      },
      regressions: [
        { type: string, url: string, before: any, after: any }
        // timing_regression also includes: threshold: number, actual: number
      ]
    }
    ```

    Export: `module.exports = { detectRegressions };`

    No external dependencies. No file I/O. No BaselineManager import — caller passes baseline object directly.
  </action>
  <verify>

          { url: '/about', title: 'About', status: 200, loadTime: 800 }

      const current = [

      const ok = result.hasRegressions === true

        && types.includes('title_changed')

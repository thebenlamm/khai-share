  <action>
    Add two requires at the top of src/routes/api.js (after existing requires):
    ```js
    const { detectRegressions } = require('../agent/regressionDetector');
    const { baselineManager } = require('./baselines');
    ```

    In the successful crawl completion block (after `const results = await crawler.close();`, before saving to completedTests):
    ```js
    // Regression detection — runs automatically when a baseline exists for this site+account
    try {
      const baseline = baselineManager.getBaselineForSite(results.site, results.account);
      results.regressions = baseline
        ? detectRegressions(baseline, results.pages)
        : null;
    } catch (err) {
      console.error('[Khai] Regression detection error:', err);
      results.regressions = null;
    }
    ```

    Place this BEFORE `completedTests.set(...)` and BEFORE `fs.writeFileSync(...)` so that:
    1. The saved report file includes regressions
    2. The completedTests map includes regressions
    3. The webhook payload (which uses `results`) includes regressions

    Do NOT add regressions to the login-failed path — login failures have no pages to compare.

    Do NOT add regressions to the error path — error path has no valid crawl data.

    The GET /api/test/:testId/results endpoint does NOT need modification — it already returns whatever is on the results object, whether from completedTests, activeTests, or the saved report file.

  <name>Task 1: Create baseline REST routes and mount in server</name>
  <files>src/routes/baselines.js, src/server.js</files>
  <action>
Create `src/routes/baselines.js` following the watches.js route pattern (module-level singleton, export router + manager).

**Endpoints:**

1. `POST /api/baselines` - Create a baseline
   - Body: `{ testId: string, thresholds?: { responseTime?: number, pageLoadTime?: number } }`
   - Validates testId is required
   - Calls `manager.createBaseline(testId, thresholds)`
   - Returns 201 with baseline object on success
   - Returns 400 if testId missing
   - Returns 404 if test report not found (catch error from manager)
   - Returns 409 if baseline already exists for that site+account (catch error from manager)

2. `GET /api/baselines` - List baselines
   - Query params: `?site=example.com` (optional filter)
   - Calls `manager.listBaselines(site)`
   - Returns `{ baselines: [...] }` (summary objects without full snapshot.pages)

3. `GET /api/baselines/:id` - Get full baseline with snapshot
   - Validates id with safeId
   - Calls `manager.getBaseline(id)`
   - Returns full baseline object including snapshot.pages
   - Returns 404 if not found

4. `PUT /api/baselines/:id` - Update baseline from new test
   - Body: `{ testId: string }`
   - Validates id with safeId, testId required
   - Calls `manager.updateBaseline(id, testId)`
   - Returns updated baseline
   - Returns 404 if baseline not found
   - Returns 404 if test report not found

5. `DELETE /api/baselines/:id` - Delete baseline
   - Validates id with safeId
   - Calls `manager.deleteBaseline(id)`
   - Returns `{ deleted: true }`
   - Returns 404 if not found

**Error handling pattern** (from BaselineManager):
- Manager methods should throw errors with descriptive messages
- Route catches errors and maps to appropriate HTTP status:
  - "not found" in message -> 404
  - "already exists" in message -> 409
  - Other errors -> errorHandler (500)

  <action>
Create `src/agent/baselineManager.js` following the WatchManager pattern (constructor loads from disk, all mutations call _save with atomic write).

**Data model** (stored in config/baselines.json):
```json
{
  "baselines": [
    {
      "id": "uuid",
      "site": "https://example.com",
      "account": "admin",
      "sourceTestId": "uuid-of-crawl-test",
      "createdAt": "ISO",
      "updatedAt": "ISO",
      "thresholds": {
        "responseTime": 5000,
        "pageLoadTime": 10000
      },
      "snapshot": {
        "capturedAt": "ISO",
        "pageCount": 12,
        "pages": [
          {
            "url": "https://example.com/page",
            "title": "Page Title",
            "status": 200,
            "loadTime": 2327
          }
        ]
      }
    }
  ]
}
```

**Constraint**: One active baseline per site+account combo (from STATE.md decisions). When creating a baseline for a site+account that already has one, return an error telling the user to update or delete the existing one.

**Default thresholds** (applied when no custom thresholds provided):
```javascript
const DEFAULT_THRESHOLDS = {
  responseTime: 5000,   // ms - individual page load time
  pageLoadTime: 10000   // ms - slow page threshold
};
```

**Methods to implement:**

1. `createBaseline(testId, thresholds = {})` - Reads the completed test report from `reports/{testId}.json`. Extracts snapshot data (pages: url, title, status, loadTime). Merges custom thresholds with defaults. Enforces one-per-site+account. Returns the created baseline object.

2. `updateBaseline(baselineId, testId)` - Reads new test report, replaces snapshot data, preserves the baseline ID and existing thresholds. Updates `updatedAt` and `sourceTestId`. Returns updated baseline.

3. `getBaseline(baselineId)` - Returns baseline or null.

4. `getBaselineForSite(site, account)` - Finds baseline by site+account combo. Returns baseline or null. Used by Phase 21 regression detection.

5. `listBaselines(site = null)` - Lists all baselines, optionally filtered by site. Returns array of baseline metadata (id, site, account, sourceTestId, createdAt, updatedAt, thresholds, snapshot.pageCount) WITHOUT full snapshot.pages (summary only).

6. `deleteBaseline(baselineId)` - Removes baseline, saves. Returns boolean.

7. `_save()` / `_load()` - Atomic persistence to `config/baselines.json` (tmp file + rename pattern from WatchManager).

**Important implementation details:**
- Use `require('uuid').v4` for baseline IDs (same as WatchManager)
- Use `require('fs')` and `require('path')` for file ops
- Path: `path.join(__dirname, '../../config/baselines.json')`
- Read reports from: `path.join(__dirname, '../../reports', testId + '.json')`
- Use safePath/safeId from `../utils/safePath` to validate testId before building file paths
- Validate that the test report exists and has pages before creating baseline
- The site URL comes from the test report's `site` field, account from `account` field

Also create an empty `config/baselines.json` with `{ "baselines": [] }` as the initial state.

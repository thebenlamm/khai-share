<task type="auto">
  <name>Task 1: Create HarRecorder utility using Chrome DevTools Protocol</name>
  <files>src/utils/har-recorder.js</files>
  <action>
Create `src/utils/har-recorder.js` that exports a `HarRecorder` class:

**Constructor:** `new HarRecorder(page)` — takes a Puppeteer Page instance.

**start() method:**
1. Create a CDP session via `this.page.target().createCDPSession()`
2. Enable `Network.enable` on the CDP session
3. Listen to these CDP events and accumulate entries:
   - `Network.requestWillBeSent` — store request start time, URL, method, headers, postData
   - `Network.responseReceived` — store status, headers, mimeType, protocol
   - `Network.loadingFinished` — store encodedDataLength; try `Network.getResponseBody` to capture body content (wrap in try/catch — some responses have no body). Mark entry as complete.
   - `Network.loadingFailed` — mark entry with error info, still include in HAR
4. Track entries in a Map keyed by requestId

**stop() method:**
1. Detach the CDP session
2. Build a HAR 1.2 JSON object from accumulated entries:
   ```json
   {
     "log": {
       "version": "1.2",
       "creator": { "name": "Khai", "version": "1.0" },
       "entries": [...]
     }
   }
   ```
3. Each entry should have: `startedDateTime` (ISO), `time` (total ms), `request` (method, url, httpVersion, headers, queryString, bodySize, postData if present), `response` (status, statusText, httpVersion, headers, content with size/mimeType/text), `timings` (send/wait/receive — derive from timestamps)
4. Return the HAR object (caller decides where to save)

**Key implementation details:**
- Use `performance.now()` style timestamps for accurate timings (Date.now() for startedDateTime, monotonic for durations)
- Response bodies: attempt `Network.getResponseBody` for each completed request. If it fails (binary, too large), set `content.text` to empty string and note `content.comment: "Body not captured"`. Limit body capture to responses < 1MB to avoid memory issues.
- Headers: CDP gives headers as objects — convert to HAR format `[{name, value}]`
- Parse query strings from URL using `new URL(url).searchParams`
- Handle data: URLs and blob: URLs gracefully (skip or minimal entry)

Modify `src/routes/actions.js` to support HAR recording:

**1. Add recordHar parameter to /execute route:**
- Destructure `recordHar = false` from `req.body` alongside existing params (line 26)
- Store `recordHar` flag in the session object in activeSessions
- Store `harFile: null` in the session object (will hold the file path when done)

**2. In the async IIFE (the background execution block starting ~line 64):**
- After `await khai.init()` and before login, if `recordHar` is true:
  - `const { HarRecorder } = require('../utils/har-recorder');`
  - `const harRecorder = new HarRecorder(khai.page);`
  - `await harRecorder.start();`
- After session completes (status = 'completed', before webhook delivery ~line 154):
  - If harRecorder exists: `const har = await harRecorder.stop();`
  - Save to disk: `const harDir = path.join(__dirname, '../../reports/har'); fs.mkdirSync(harDir, { recursive: true }); const harPath = path.join(harDir, sessionId + '.har'); fs.writeFileSync(harPath, JSON.stringify(har, null, 2));`
  - Set `session.harFile = harPath;`
- In error/login-failed terminal states: also call `harRecorder.stop()` (if it exists) and save — partial traces are still valuable
- Add `const path = require('path');` at top if not already imported (it is not currently imported in actions.js)

**3. Add harFile to response payloads:**
- In GET /status/:sessionId response: add `harFile: session.harFile || null`
- In GET /results/:sessionId response: add `harFile: session.harFile || null`

**4. Add new GET /har/:sessionId endpoint (before the quick action endpoints ~line 224):**
```javascript
router.get('/har/:sessionId', (req, res) => {
  const { sessionId } = req.params;
  const session = activeSessions.get(sessionId);

  if (!session) {
    return res.status(404).json(fail('Session not found'));
  }

  if (!session.harFile) {
    return res.status(404).json(fail('No HAR file for this session. Was recordHar enabled?'));
  }

  // Serve the HAR file with correct content type
  res.setHeader('Content-Type', 'application/json');
  res.setHeader('Content-Disposition', `attachment; filename="${sessionId}.har"`);
  const harStream = require('fs').createReadStream(session.harFile);
  harStream.pipe(res);
});
```

**5. Update start response:** If recordHar is true, include `recordHar: true` in the start response object (~line 177).

**6. Include webhook payloads:** Add `harFile: session.harFile || null` to all three webhook delivery payloads (completed, error, login-failed).

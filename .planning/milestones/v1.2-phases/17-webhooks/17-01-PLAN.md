<task type="auto">
  <name>Task 1: Create webhook delivery utility</name>
  <files>src/utils/webhook.js</files>
  <action>
Create `src/utils/webhook.js` exporting a single async function `deliverWebhook(webhookUrl, payload, options)`.

The function must:

1. **Accept parameters:**
   - `webhookUrl` (string) — target URL to POST to
   - `payload` (object) — the full operation results to send as JSON body
   - `options.secret` (string, optional) — HMAC shared secret. Read from `process.env.KHAI_WEBHOOK_SECRET` if not provided. If neither exists, skip signing but still deliver.
   - `options.operationType` (string) — e.g. "test", "audit", "action", "link-check"
   - `options.operationId` (string) — the testId/auditId/sessionId/jobId

2. **Sign the payload** with HMAC-SHA256:
   - Use Node.js built-in `crypto` module (no new dependencies)
   - Compute: `crypto.createHmac('sha256', secret).update(bodyString).digest('hex')`
   - Send as `X-Khai-Signature` header (format: `sha256=<hex>`)
   - If no secret is available, omit the signature header

3. **POST the payload** using Node.js built-in `http`/`https` modules (no new dependencies):
   - Headers: `Content-Type: application/json`, `User-Agent: Khai/1.0`, `X-Khai-Event: operation.completed`, `X-Khai-Operation: {operationType}`, `X-Khai-Operation-Id: {operationId}`
   - Timeout: 10 seconds per attempt
   - Consider any 2xx status code as success

4. **Retry on failure** up to 3 times with exponential backoff:
   - Delays: 1s, 4s, 16s (base 1s, multiplier 4x)
   - Retry on: network errors, 5xx responses, timeouts
   - Do NOT retry on: 4xx responses (client error = permanent failure)

5. **Return a status object:**
   ```js
   {
     status: 'delivered' | 'failed',
     url: webhookUrl,
     attempts: N,
     lastAttemptAt: ISO timestamp,
     statusCode: HTTP status code or null,
     error: error message or null
   }
   ```

6. **Never throw** — always return the status object. Log errors with `console.error('[Khai] Webhook ...')`.

Do NOT use any external HTTP library (no axios, node-fetch, etc.). Use Node.js built-in `http`/`https` modules, same pattern as the existing `_postWebhook` in `src/agent/scheduler.js`.

<task type="auto">
  <name>Task 2: Integrate webhooks into all async operation routes</name>
  <files>src/routes/api.js, src/routes/actions.js, src/routes/audit.js, src/routes/advanced.js</files>
  <action>
Integrate `deliverWebhook` from `src/utils/webhook.js` into the four async operation route files. Each integration follows the same pattern:

**Pattern for each route:**

1. Add `const { deliverWebhook } = require('../utils/webhook');` at top
2. Extract `webhookUrl` from the request body in the start endpoint
3. Store `webhookUrl` on the active operation object (in the Map)
4. After the operation completes (status set to 'completed', 'error', or 'login-failed'), call `deliverWebhook` if `webhookUrl` was provided
5. Store the webhook delivery status on the operation object as `webhook` field
6. Include the `webhook` field in status and results responses

**Specific integrations:**

**src/routes/api.js — POST /api/test/start:**
- Extract `webhookUrl` from `req.body` (alongside existing site, account, etc.)
- Store on the test object in `activeTests`
- In the async IIFE, after `test.status = 'completed'` (line ~131) AND after `test.status = 'login-failed'` (line ~110) AND after `test.status = 'error'` (line ~143), add:
  ```js
  if (test.webhookUrl) {
    const results = completedTests.get(testId) || test.crawler?.results;
    test.webhook = await deliverWebhook(test.webhookUrl, results, {
      operationType: 'test', operationId: testId
    });
  }
  ```
- In the status endpoint (GET /test/:testId/status), include `webhook: test.webhook || null` in the response
- In the start response, include `webhookUrl` if provided (for confirmation)

**src/routes/actions.js — POST /api/actions/execute:**
- Extract `webhookUrl` from `req.body`
- Store on session object in `activeSessions`
- After `session.status = 'completed'` (line ~145) AND after `session.status = 'login-failed'` (line ~74) AND after `session.status = 'error'` (line ~150), add webhook delivery:

    session.webhook = await deliverWebhook(session.webhookUrl, {
      sessionId, status: session.status, results: session.results,
      startTime: session.startTime, error: session.error || null
    }, { operationType: 'action', operationId: sessionId });
  }
  ```
- Include `webhook: session.webhook || null` in status and results responses

**src/routes/audit.js — POST /api/audit/start:**
- Extract `webhookUrl` from `req.body`

- After `status = 'completed'` (line ~123) AND after `status = 'error'` (line ~127), add webhook delivery:

      operationType: 'audit', operationId: auditId
    });
  }
  ```
- Include `webhook: audit.webhook || null` in status response

**IMPORTANT:** The webhook delivery call must be `await`ed but must NOT block or throw — `deliverWebhook` never throws by design. Place it AFTER the status is set and results are saved, so the operation is already complete when the webhook fires.

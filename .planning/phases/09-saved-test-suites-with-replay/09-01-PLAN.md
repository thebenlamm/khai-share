```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://khai.local/schemas/suite.json",
  "title": "Khai Test Suite",
  "description": "Schema for Khai test suite manifest files",

    "suite": {
      "type": "object",
      "required": ["id", "name", "version"],
      "properties": {
        "id": {
          "type": "string",
          "pattern": "^[a-z0-9-]+$",
          "description": "Unique suite identifier (kebab-case)"
        },
        "name": {
          "type": "string",
          "minLength": 1,
          "description": "Human-readable suite name"
        },
        "version": {
          "type": "string",
          "pattern": "^\\d+\\.\\d+\\.\\d+$",
          "description": "Semantic version"
        },
        "tags": {
          "type": "array",
          "items": { "type": "string" },
          "uniqueItems": true,
          "description": "Suite-level tags"
        },
        "environment": {
          "type": "string",
          "enum": ["staging", "production", "local"],
          "description": "Target environment"
        },
        "timeout": {
          "type": "integer",
          "minimum": 1000,
          "maximum": 3600000,
          "default": 300000,
          "description": "Suite timeout in milliseconds"
        },
        "created": {
          "type": "string",
          "format": "date-time"
        },
        "updated": {
          "type": "string",
          "format": "date-time"
        }
      }
    },
    "tests": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "object",
        "required": ["type"],
        "properties": {
          "type": {
            "type": "string",
            "enum": ["auth", "performance", "visual", "dry-run", "animation"],
            "description": "Test type"
          },
          "role": {
            "type": "string",
            "enum": ["admin", "agent", "seller", "buyer"],
            "description": "HomeBay role for auth/performance/visual tests"
          },
          "config": {
            "type": "object",
            "description": "Test-specific configuration"
          },
          "tags": {
            "type": "array",
            "items": { "type": "string" },
            "uniqueItems": true,
            "description": "Test-level tags for filtering"
          },
          "critical": {
            "type": "boolean",
            "default": false,
            "description": "Fail entire suite if this test fails"
          }
        }
      }
    }
  }
}
```

This schema enforces:
- Required fields (suite.id, suite.name, suite.version, tests array)
- Valid test types (auth, performance, visual, dry-run, animation)
- Valid roles (admin, agent, seller, buyer)
- Timeout bounds (1s - 1 hour)
- Semantic versioning pattern
- Kebab-case suite IDs

Implementation details:
- **Schema validation**: Use ajv with ajv-formats for date-time validation. Validate in constructor before storing suite. Throw descriptive error if invalid (SUITE-01).
- Suite timeout: Use Promise.race with setTimeout reject (default 300000ms from suite.suite.timeout or schema default)
- Tag filtering: If this.tags non-empty, skip tests where !this.tags.some(t => test.tags?.includes(t))
- Test execution: Switch statement on test.type → call loginHomeBay(role), auditHomeBayRole(role), captureHomeBayRole(role), etc.
- Critical failures: If test.critical===true and result.status==='failed', throw error to fail-fast
- Result structure: Each test result includes { ...test, status: 'passed'|'failed'|'skipped', duration, resultPath?, error? }
- History log: Newline-delimited JSON with one entry per suite run: {runId, suiteId, status, duration, passRate}

**Why ajv not JSON.parse()**: JSON.parse() only checks JSON syntax. ajv validates against schema (required fields, valid enums, type constraints). Per research line 38-41 and SUITE-01 requirement "JSON schema validation", must use ajv.

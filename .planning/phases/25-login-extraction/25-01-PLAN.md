Create src/utils/login.js exporting a single async function:

```js
async function performLogin(page, baseUrl, accountConfig, options = {})
```

Parameters:
- `page` — Puppeteer Page object
- `baseUrl` — site base URL (e.g. "https://example.com")
- `accountConfig` — credential config object (shape documented in interfaces above)
- `options` — optional: `{ screenshotFn, logger, addIssueFn }`
  - `screenshotFn(name)` — called for screenshot capture if provided (agents pass their own screenshot method)
  - `logger` — defaults to console; agents can pass a prefixed logger
  - `addIssueFn(type, message, url, severity)` — called when login fails; only crawler uses this currently

Returns: `{ success: boolean, error?: string }`

Implementation — consolidate all 5 auth paths in this order (matching crawler.js logic):

1. **Magic link** (`accountConfig.magicLinkAuth` is truthy):
   - `const fetch = (await import('node-fetch')).default;`
   - Fetch the magic link URL, extract `data.loginUrl`
   - Navigate to `data.loginUrl`, wait 4000ms for redirect
   - Screenshot 'magic-link-login'
   - Check if still on `/login` without `/token` — return `{ success: false, error: 'Magic link login failed' }`
   - Otherwise return `{ success: true }`

2. **Skip login** (`accountConfig.skipLogin` is truthy):
   - Navigate to `baseUrl`, wait 2000ms
   - Screenshot 'public-start'
   - Return `{ success: true }`

3. **Standard/Twilio/Trigger flow** (all others):
   a. Navigate to `baseUrl + accountConfig.loginUrl`, wait 3000ms for SPA hydration
   b. Screenshot 'login-page'
   c. **Twilio detection**: if `page.url().includes('twilio.com')`, call internal `_loginTwilio(page, accountConfig, options)` — extract from actions.js lines 135-253 verbatim
   d. **Login trigger**: if `accountConfig.loginTrigger`, use the `page.evaluate` pattern from actions.js lines 47-68 to find button by has-text and click it, wait 2000ms
   e. **Fill email**: iterate `accountConfig.usernameField.split(',')`, try `page.waitForSelector` with 5000ms timeout, then `page.type` with `{ delay: 50 }`. If no selector works, use crawler.js fallback: `page.evaluate` to find visible `input[type="email"]` or `input[type="text"]`.
   f. **Fill password**: iterate `accountConfig.passwordField.split(',')`, try each with 3000ms timeout, `page.type` with `{ delay: 50 }`. Fallback to `input[type="password"]`.
   g. **Submit**: iterate `accountConfig.submitButton.split(',')`, click first found button with `Promise.all([btn.click(), page.waitForNavigation()])`. If none found, fallback `page.keyboard.press('Enter')` + waitForNavigation.
   h. Wait 2000ms, screenshot 'after-login'
   i. **Verify success**: check if `page.url()` still includes the login path and not 'callback' — return `{ success: false, error: 'Still on login page after submit' }` if so
   j. Return `{ success: true }`

4. Wrap entire function body in try/catch. On error: screenshot 'login-error', return `{ success: false, error: err.message }`.

Internal helper `_loginTwilio(page, accountConfig, options)` — extract from actions.js lines 135-253:
- Step 1: Find email input (placeholder Email, type email, name email, autocomplete email, fallback first visible input), click + type username
- Step 2: Find and click Continue/Next button via page.evaluate
- Step 3: Wait for password input, fill it
- Step 4: Find and click Login/Sign In/Continue button
- Return `{ success: true }` (matching current behavior of always returning true)

Use `options.screenshotFn?.(name)` for all screenshots (no-op if not provided).
Use `options.logger?.log?.()` or `console.log()` for logging with prefix `[Login]`.
Use `options.addIssueFn?.()` for issue reporting (only called on failure for crawlers).

Export: `module.exports = { performLogin };`

  <action>
Create `config/homebay-dryrun.json` defining test cases for critical HomeBay forms. Each test case specifies the form URL, fields to fill, and expected validation errors.

**File structure:**

```json
{
  "register": {
    "url": "/register",
    "role": "buyer",
    "tests": [
      {
        "name": "Missing required fields",
        "formData": {
          "email": "test@example.com",
          "password": ""
        },
        "expectedErrors": ["Password is required"]
      },
      {
        "name": "Invalid email format",
        "formData": {
          "email": "notanemail",
          "password": "ValidPass123!"
        },
        "expectedErrors": ["Please enter a valid email address"]
      },
      {
        "name": "Password too short",
        "formData": {
          "email": "test@example.com",
          "password": "short"
        },
        "expectedErrors": ["Password must be at least 8 characters"]
      }
    ]
  },
  "login": {
    "url": "/login",
    "role": null,
    "tests": [
      {
        "name": "Missing email",
        "formData": {
          "email": "",
          "password": "password123"
        },
        "expectedErrors": ["Email is required"]
      },
      {
        "name": "Missing password",
        "formData": {
          "email": "test@example.com",
          "password": ""
        },
        "expectedErrors": ["Password is required"]
      }
    ]
  },
  "forgot-password": {
    "url": "/forgot-password",
    "role": null,
    "tests": [
      {
        "name": "Missing email",
        "formData": {
          "email": ""
        },
        "expectedErrors": ["Email is required"]
      },
      {
        "name": "Invalid email format",
        "formData": {
          "email": "notanemail"
        },
        "expectedErrors": ["Please enter a valid email address"]
      }
    ]
  }
}
```

**Field explanations:**
- `url`: Relative path to the form page (e.g., `/register`, `/login`)
- `role`: User role to authenticate as before testing (null = no auth required, "buyer"/"admin"/etc. = login first)
- `tests[]`: Array of test cases
  - `name`: Human-readable test description
  - `formData`: Object mapping input field IDs to values (e.g., `{ email: "...", password: "..." }`)
  - `expectedErrors`: Array of expected validation error messages (React or HTML5)

**Test coverage rationale:**
- **register**: Tests client-side validation for required fields, email format, password strength
- **login**: Tests required field validation (no auth needed since this IS the login form)
- **forgot-password**: Tests email validation on password reset flow

**Why these forms:** Phase 1 already has auth flows implemented (loginHomeBay, registerHomeBay, forgotPasswordHomeBay). Dry-run tests validate the client-side validation logic for these critical entry points without creating test accounts or sending emails.

**Future expansion:** Phase 2+ can add test cases for bid forms, auction creation forms, payment forms, etc. as those workflows are implemented.

<task type="auto">
  <name>Task 2: Add POST /api/homebay/dryrun/:form endpoint</name>
  <files>src/routes/homebay.js</files>
  <action>
Add `POST /api/homebay/dryrun/:form` endpoint to `src/routes/homebay.js` following Khai's existing API patterns.

**Endpoint behavior:**

1. **Route definition:**
   ```javascript
   router.post('/dryrun/:form', async (req, res) => { ... });
   ```

2. **Load test configuration:**
   ```javascript
   const fs = require('fs');
   const path = require('path');

   const configPath = path.join(__dirname, '../../config/homebay-dryrun.json');
   if (!fs.existsSync(configPath)) {
     return res.status(404).json(fail('Dry-run config not found: config/homebay-dryrun.json'));
   }

   const dryrunConfig = JSON.parse(fs.readFileSync(configPath, 'utf8'));
   const formName = req.params.form;
   const formConfig = dryrunConfig[formName];

   if (!formConfig) {
     return res.status(404).json(fail(`Form '${formName}' not found in dry-run config`));
   }
   ```

3. **Authentication (if required):**
   ```javascript
   const { loginHomeBay } = require('../homebay/auth');

   if (formConfig.role) {
     console.log(`[Khai] Dry-run test for ${formName} requires auth as ${formConfig.role}`);
     const authResult = await loginHomeBay(formConfig.role);
     if (!authResult.success) {
       return res.status(400).json(fail(`Auth failed: ${authResult.error}`));
     }
   }
   ```

4. **Run all test cases:**
   ```javascript
   const { DryRunTester } = require('../homebay/dryrun');
   const { getHomeBayConfig } = require('../homebay/config');
   const config = getHomeBayConfig();

   const results = [];
   for (const test of formConfig.tests) {
     const tester = new DryRunTester({ baseUrl: config.baseUrl });
     const result = await tester.testFormValidation(
       formConfig.role,
       formConfig.url,
       test.formData,
       test.expectedErrors
     );
     results.push({
       name: test.name,
       passed: result.passed,
       html5Valid: result.html5Valid,
       reactErrors: result.reactErrors,
       expectedErrors: test.expectedErrors
     });
   }
   ```

5. **Response format:**
   ```javascript
   const passCount = results.filter(r => r.passed).length;
   const failCount = results.length - passCount;

   return res.json(success({
     form: formName,
     totalTests: results.length,
     passed: passCount,
     failed: failCount,
     results
   }));
   ```

6. **Error handling:**
   - Wrap entire handler in try/catch
   - Log errors with `[Khai]` prefix: `console.error('[Khai] Error in /api/homebay/dryrun:', error)`
   - Return 500 with generic message: `res.status(500).json(fail('Internal server error'))`

**Integration points:**
- Import DryRunTester from `../homebay/dryrun`
- Import loginHomeBay from `../homebay/auth` for authenticated forms
- Import getHomeBayConfig from `../homebay/config` for baseUrl
- Import response utilities: `const { success, fail } = require('../utils/response')`

**Add at bottom of src/routes/homebay.js** after existing endpoints (login, login/all, register, forgot-password, reset-password, health, config).

**Why synchronous execution:** Dry-run tests are fast (<5 seconds per form). No need for async job pattern like crawl/audit operations. Return results immediately in the POST response.

**Testing note:** The research's Validation Architecture section notes that HomeBay is a testing tool, so testing the tests is optional. Unit tests for DryRunTester would require mocking Puppeteer page objects, which adds complexity without user value. Manual smoke testing via curl is sufficient.

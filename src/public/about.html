🔋 Powers (What Khai CAN Do)

  • 🔐

    Automated Login

    Log into websites using stored credentials at admin and patient levels. Navigate authenticated areas.

  • 🕷️

    Deep Crawling

    Discover and visit every link on a website up to a configurable depth. Map the entire site structure.

  • 📸

    Screenshot Capture

    Take full-page screenshots of every page visited for visual documentation and debugging.

  • 🔍

    Issue Detection

    Identify HTTP errors, slow pages, broken images, JavaScript errors, accessibility issues, and security warnings.

  • 💳

    Purchase Testing

    Fill payment forms with stored card details. Requires your confirmation before completing any purchase.

  • 📧

    Communication Monitoring

    Monitor incoming emails, SMS, and faxes. Auto-extract verification codes for 2FA flows.

  • 📝

    Notes & Reports

    Add notes to issues. Export comprehensive JSON reports for debugging and handoff.

⚠️ Limitations (What Khai CANNOT Do)

  • 🚫

    No Autonomous Purchases

    Khai will NEVER complete a purchase without your explicit confirmation. You always have final say.

  • 🔒

    No Code Modification (Report Only)

    Khai detects issues but does not modify your website's source code. All fixes are manual.

  • 🌐

    Local Only

    Runs on localhost only. Not exposed to the internet. Your credentials stay on your machine.

  • ⏱️

    No Real-Time Alerts

    Must be actively running to monitor. Does not run as a background service or send push notifications.

  • 🤖

    No CAPTCHA Solving

    Cannot bypass CAPTCHAs. Pages with CAPTCHA protection may require manual intervention.

  • 📱

    Desktop Viewport Only

    Tests at 1920x1080 resolution. Does not automatically test mobile or tablet viewports.

🛡️ Security Measures

  • 🔐

    Credentials Never Exposed

    Stored in local config file. Never logged, never transmitted. Add credentials.json to .gitignore.

  • ✅

    Purchase Confirmation Required

    All payment actions require explicit user approval through the UI before execution.

  • 🏠

    Localhost Binding

    Server only accessible from your machine. Not exposed on network interfaces.

  • 📋

    Audit Trail

    All actions logged with timestamps. Reports saved for review. Screenshots document state.

⚖️ Legal Constraints & Reminders

  • 📜

    Authorization Required

    Only test websites you OWN or have EXPLICIT WRITTEN PERMISSION to test. Unauthorized testing may violate the Computer Fraud and Abuse Act (CFAA) and similar laws.

  • 🏥

    HIPAA Compliance

    For healthcare sites: Ensure testing doesn't expose PHI. Screenshots may contain patient data. Store securely and delete when no longer needed.

  • 💳

    PCI-DSS Awareness

    Payment card data in config files must be protected. Use test cards when possible. Never commit credentials.json to version control.

  • 📊

    Data Retention

    Screenshots and reports may contain sensitive data. Implement a retention policy. Delete old test data regularly.

  • 🌍

    GDPR/CCPA

    Testing may process personal data. Ensure compliance with applicable privacy regulations for data you collect during tests.

  • 📝

    Terms of Service

    Automated testing may violate some websites' ToS. Review terms before testing third-party services.

⏰ Operational Reminders

  • 🔄

    Rotate Credentials Regularly

    Change passwords stored in config periodically. Use dedicated test accounts when possible.

  • 👁️

    Review Before Purchase Confirmation

    Always verify the amount, product, and card before confirming any purchase. Test with minimal amounts first.

  • 🗑️

    Clean Up Test Data

    After testing, clean up any test accounts, orders, or data created on the live site.

  • 📅

    Schedule During Low Traffic

    Run comprehensive tests during off-peak hours to minimize impact on real users.

  • 📞

    Have Rollback Plan

    Know how to quickly revert any changes if testing causes unexpected issues.

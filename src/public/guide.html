Overview

Khai is a local browser-automation server for cases where Claude needs live website access. The Express server runs on 127.0.0.1:3001. The MCP server exposes the most common operations as Claude Code tools.

Authenticated crawling

Log in with stored credentials, crawl protected pages, collect screenshots, and flag issues such as HTTP failures, redirects back to login, slow pages, resource failures, and accessibility problems.

Targeted browser actions

Run focused sequences against a logged-in session. Current action types are navigate, wait, screenshot, evaluate, create-note, send-fax, send-sms, and twilio-a2p.

Operational monitoring

Create watches for authenticated pages, store watch history, trigger webhooks on changes, and maintain crawl baselines for later regression comparison.

REST-only extras

Suites, Lighthouse, flow testing, visual comparison, fuzzing, password rotation, communications monitoring, and scheduler endpoints are available through the Express API even when they are not exposed as MCP tools.

Quick Start

# install npm install pip install -e . # configure sites cp config/credentials.example.json config/credentials.json # start the Express server npm start # optional: start the MCP server khai-mcp # or SSE mode khai-mcp --sse
Important: the Express server must be running before the MCP tools will work. The local UI lives at /, this guide is /guide.html, and the safety notes are /about.html.

Crawl Tests

Crawl tests are the main UI workflow. A test logs in using a configured account, visits internal pages up to a chosen depth, records screenshots, and stores a JSON report under reports/.

Current crawl viewport presets in the server are desktop, tablet, and mobile. The MCP wrapper currently documents desktop and mobile; the REST API accepts whatever the crawler recognizes.
RouteUse
POST /api/test/startStart a crawl with {site, account, maxDepth, viewport?, startPath?, webhookUrl?}
GET /api/test/:testId/statusPoll current status, page count, issue count, summary, pending purchase, and webhook state
GET /api/test/:testId/resultsFetch the full results payload
POST /api/test/:testId/stopStop a running crawl
DELETE /api/test/:testIdDelete report and screenshots for a crawl
GET /api/testsList active and saved crawl runs

Action Sessions

Action sessions are for targeted work after login. They are asynchronous like crawl tests and can optionally record a HAR trace.

Action typeNotes
navigateOpen a relative path or same-host URL. Cross-host navigation is blocked unless KHAI_ALLOW_EXTERNAL_NAV=true.
waitPause for a duration in milliseconds.
screenshotCapture a named screenshot.
evaluateRun page JavaScript. Disabled unless KHAI_ALLOW_EVAL=true.
create-noteProject-specific helper action implemented in the action agent.
send-faxProject-specific helper action implemented in the action agent.
send-smsProject-specific helper action implemented in the action agent.
twilio-a2pProject-specific helper action implemented in the action agent.

Watches and Baselines

Watches are persistent scheduled checks for authenticated pages. Baselines are snapshots from completed crawl tests used for regression comparison.

Watch cron expressions are scheduled in UTC. Webhook notifications fire only when a watch detects a change.

WatchesPOST /api/watches, GET /api/watches, GET /api/watches/:id, PUT /api/watches/:id, DELETE /api/watches/:id, GET /api/watches/:id/history, POST /api/watches/:id/run

REST Reference

The Express app serves both the local UI and the HTTP API. If KHAI_API_KEY is set, requests under /api/* require an X-Khai-Key header.

# health check curl http://127.0.0.1:3001/health # list configured sites curl http://127.0.0.1:3001/api/sites # create a watch curl -X POST http://127.0.0.1:3001/api/watches \ -H "Content-Type: application/json" \ -d '{"site":"yoursite.com","account":"admin","url":"/dashboard","schedule":"0 * * * *"}'

MCP Tools

The MCP layer exposes the most common Express workflows to Claude Code. Use MCP for site discovery, crawls, action sessions, audits, link checks, watches, and baselines.

MCP parameters use snake_case. Example: webhook_url, record_har, max_depth, start_path. The REST API uses camelCase instead.
